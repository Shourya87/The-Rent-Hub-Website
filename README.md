## ğŸ—ƒï¸ Database Setup (Postgres - Recommended)

You can run in two modes:

1. **Postgres mode (recommended for production)**
2. **JSON mode fallback (good for local quick start)**

### A) Postgres mode (using Supabase Postgres)

Create table in Supabase SQL editor:

```sql
create table if not exists public.properties (
  id bigint generated by default as identity primary key,
  title text,
  location text,
  price numeric,
  beds int,
  baths int,
  area int,
  image text,
  video text,
  highlights jsonb,
  description text,
  featured boolean default false,
  bhk text,
  category text,
  propertyType text,
  details jsonb,
  createdAt timestamptz default now(),
  updatedAt timestamptz default now()
);
```

Set backend env variables:

- `SUPABASE_URL=https://<project-ref>.supabase.co`
- `SUPABASE_SERVICE_ROLE_KEY=<service-role-key>`
- `SUPABASE_PROPERTIES_TABLE=properties` (optional, default `properties`)

### B) JSON mode fallback

If `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` are not set, backend auto-uses:

- `backend/src/data/properties.json`

## ğŸš€ Deployment Guide

## 1) Backend deploy (Render recommended)

- Create **Render Web Service**
- Root directory: `backend`
- Build command: `npm install`
- Start command: `npm start`
- Set env vars listed above
- Add persistent disk if using local uploads in production

## 2) Database deploy

- Use Supabase Postgres (recommended)
- Run SQL table creation
- Add `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` to Render backend

## 3) Frontend deploy (Vercel recommended)

- Root directory: `frontend`
- Build command: `npm run build`
- Output directory: `dist`
- Env vars:
  - `VITE_API_BASE_URL=https://<your-backend-domain>`
  - optional `VITE_PROXY_TARGET` for local/dev only

## 4) CORS setup

Set backend `FRONTEND_ORIGIN` to your deployed frontend domain(s):

```txt
https://your-frontend.vercel.app,https://www.yourdomain.com
```
